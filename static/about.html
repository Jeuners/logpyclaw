Das Projekt

Was ist AgentClaw?

AgentClaw ist ein lokal laufendes Multi-Agent-System, das auf dem eigenen Mac läuft — ohne Cloud, ohne externe Abhängigkeiten. Es verbindet mehrere KI-Agenten, die Aufgaben untereinander delegieren, Ergebnisse prüfen und dabei ihre eigene „Zeitwahrnehmung" tracken.

Der Kern des Projekts ist die Erforschung der Frage: Wie nimmt ein KI-Agent Zeit wahr? Nicht Wanduhrzeit — sondern Eigenzeit, definiert durch die Anzahl der verarbeiteten Operationen. Daraus entsteht das Konzept der Causal-Dilation Clock (CDC).

🤖

Mock-Agenten

Spawnable Agenten mit verschiedenen Policies — von schnell bis absichtlich langsam, mit Fehlerwahrscheinlichkeit und QC-Überprüfung.

⏱

Causal-Dilation Clock

Jede Message trägt eine Uhr, die kausale Reihenfolge und die Eigenzeit-Rate jedes Agenten erfasst — als (V, D)-Tupel.

🎬

LTX Video Batch

WAV + Bild → Segmentierung → Prompts via Ollama → Video-Rendering via ComfyUI LTX 2.3, mit Frame-Chaining zwischen Segmenten.

🌌

Spacetime-Visualisierung

Interaktives Diagramm: Agenten als Weltlinien, Messages als Pfeile, Drift-Erkennung farblich codiert — direkt aus den Live-Daten.

Ehrliche Einschätzung

Stärken und Schwächen

AgentClaw ist kein fertiges Produkt — es ist ein experimentelles System. Hier ist, was funktioniert und was noch nicht.

✓ Stärken
✓
Vollständig lokal — keine Cloud, keine Daten verlassen den Mac. Volle Kontrolle über Modelle und Daten.
✓
Neuartiges Zeitkonzept — die CDC bringt echte Forschungsinnovation: Eigenzeit für KI-Agenten als messbares Konzept.
✓
Modulare Architektur — Lab, LTX-Batch, Chat, API sind klar getrennt. Neue Module lassen sich isoliert entwickeln.
✓
QC-Doppel-Agenten — automatische Qualitätsprüfung via Reviewer-Agent mit Score, Retry-Logik und konfigurierbarer Rate.
✓
Drift-kompensierter Scheduler — Agenten-Auswahl basiert auf gemessener Eigenzeit-Rate (γ_ij), nicht auf Heuristiken.
✓
Video-Produktion — vollständige Pipeline von Audio zu Video, inkl. Frame-Chaining und nachträglichem Concat.
✗ Schwächen
✗
Kein Neustart-Persistenz im Lab — Missionen und Agenten gehen nach Server-Neustart verloren. Kein JSONL-Replay für das Lab.
✗
NiceGUI/Python-3.14-Bug — alle UI-Updates müssen client-seitig in JavaScript erfolgen. Schränkt Entwicklungskomfort ein.
✗
Keine echten LLM-Agenten — das Lab nutzt Mock-Agenten. Echte LLM-Calls sind im Chat, aber nicht tief ins Agent-Protokoll integriert.
✗
Hohe Hardware-Anforderungen — Ollama, ComfyUI, Redis, Qdrant — alles lokal. Mindestens 32 GB RAM und GPU empfohlen.
✗
Keine Authentifizierung — das System hat keine Nutzer-Verwaltung, keine Rollen, kein Auth. Nur für lokalen Einzelbetrieb.
✗
Kein UI-Onboarding — wer das System nicht selbst gebaut hat, findet sich kaum zurecht. Null Dokumentation für Endnutzer.
Wichtiger Hinweis

Das ist (noch) kein Consumer-Produkt

AgentClaw ist ein aktives Forschungs- und Experimentierprojekt. Es wird täglich verändert, bricht manchmal, und setzt technisches Verständnis voraus. Es gibt keine stabile Version, kein Installer, keinen Support.

⚠️ Was das bedeutet

  • Kein Installer — Inbetriebnahme erfordert Python, Ollama, ComfyUI, Redis, Qdrant manuell einzurichten.
  • Keine Stabilität — Breaking Changes passieren ohne Ankündigung oder Migration.
  • Keine Dokumentation für Endnutzer — nur Code-Kommentare und dieses Projekt-CLAUDE.md.
  • Keine Sicherheit — kein Auth, kein Rate-Limiting, nicht für öffentliche Server geeignet.
  • Keine Garantien — Modell-Outputs sind experimentell, Ergebnisse nicht reproduzierbar.
  • Kein Support — wer es nutzt, muss es selbst verstehen und anpassen können.

Das Ziel ist, Konzepte zu erforschen, Ideen auszuprobieren und die Grenzen von Multi-Agent-Systemen zu verstehen — nicht, ein fertiges Produkt zu liefern. Wenn daraus eines Tages etwas Produktreifes entsteht, wird das hier dokumentiert.
